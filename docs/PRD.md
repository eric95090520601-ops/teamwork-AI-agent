# PRD: 租屋網站 - 房東基本資料透明化(真房東認證)功能

## 1. 專案概述
- **背景與動機**：近年租屋詐騙頻傳，假房東騙取押金事件層出不窮。為了提升平台信任度，保障租客權益，平台需建立「真房東」認證機制，確保物件刊登者具備合法出租資格。
- **目標用戶**：房東（上傳認證資料）、租客（查看認證標章）、平台管理員（審核認證資料）。
- **核心價值主張**：透過實名認證與房屋產權查驗，過濾不實房源與詐騙份子，打造安心、透明的租屋環境。

## 2. 功能需求
- 作為 **房東**，我希望 **在刊登物件前能上傳身分證與房屋權狀影本（或仲介證照）**，以便 **申請真房東認證**。
- 作為 **房東**，我希望 **能在個人介面追蹤認證審核進度（如：審核中、已通過、已退件）**，以便 **隨時掌握認證狀態**。
- 作為 **平台管理員**，我希望 **有一個後台介面能審核房東上傳的認證文件**，以便 **確認其真實性並核發或拒絕認證**。
- 作為 **系統**，我希望 **能在房東通過認證後，自動在其刊登的物件上顯示「真房東認證」標章**，以便 **讓租客辨識**。
- 作為 **租客**，我希望 **在瀏覽房屋物件時，能清楚看到房東是否通過「真房東認證」**，以便 **作為租屋決策的參考**。

## 3. 非功能需求
- **技術限制**：使用 Flask + Jinja2 + SQLite。
- **效能與安全考量**：
  - 上傳的身分證與權狀影本屬於高度敏感個資，必須安全儲存，且除了審核管理員外不可對外公開。
  - 上傳圖片需限制檔案大小與格式（如 JPG, PNG, PDF 等）。
  - 自動在圖片上壓浮水印（例如「僅供租屋平台審核使用」），防止個資遭盜用。

## 4. MVP 範圍
- **Must Have**: 
  - 房東上傳身分證與房屋權狀/仲介證照介面。
  - 平台管理員審核介面（核准/拒絕）。
  - 認證狀態關聯到房東資料。
  - 前台物件頁面與房東資料顯示「真房東認證」標章。
- **Should Have**:
  - 審核退件時可填寫退件理由。
  - 房東上傳文件時，系統自動加上防偽浮水印（保護隱私）。
- **Nice to Have**: 
  - 介接政府 API 進行自動驗證（取代人工審核）。
  - 認證狀態變更時發送 Email/系統通知給房東。

# PRD - 反詐租屋網：線上繳交房租功能

## 1. 專案概述

- **背景與動機**：在許多租屋詐騙案例中，租客常因匯款帳戶不明、無正式繳費紀錄而蒙受損失。「反詐租屋網」旨在建立一個安全的租屋生態，而「線上繳交房租與紀錄查詢」是保障雙方權益的核心功能。透過平台紀錄每一筆金流，可大幅降低詐騙風險。
- **目標用戶**：已經在平台完成簽約手續的租客。
- **核心價值主張**：提供安全透明的繳費管道，並永久保存繳費紀錄，確保租客的每一分錢都有跡可循。

## 2. 功能需求

以下是本模組的主要功能需求：

1. **查看當期帳單**
   - 作為 **租客**，我希望 **登入後能看到本月應繳的房租金額與期限**，以便 **準時繳交房租**。
2. **線上繳費 (模擬信用卡/匯款)**
   - 作為 **租客**，我希望 **能在網站上輸入繳費資訊（如信用卡號或上傳匯款證明）進行繳費**，以便 **快速完成繳租**。
3. **安全提示與防詐確認**
   - 作為 **租客**，我希望 **在送出繳費前，系統能跳出防詐騙安全提示（確認收款方資訊）**，以便 **確保不會匯錯對象**。
4. **查看繳費紀錄**
   - 作為 **租客**，我希望 **能查看過去所有的歷史繳費紀錄與狀態（成功/處理中/失敗）**，以便 **日後備查與對帳**。
5. **產生假資料（開發與展示用）**
   - 作為 **開發者/測試人員**，我希望 **系統能自動載入一組假會員與假租約資料**，以便 **直接測試繳費流程而無需從頭註冊**。

## 3. 非功能需求

- **技術限制**：
  - 後端使用 Python + Flask。
  - 網頁渲染使用 Jinja2 模板引擎（無前後端分離）。
  - 資料庫使用 SQLite。
- **效能與安全考量**：
  - 介面設計需給予使用者「安全、可信賴」的感受（如：防詐警告標示、SSL 鎖頭圖示感）。
  - （模擬）信用卡資訊傳輸應妥善處理，前端需加入基本的表單驗證。
  - 採用現代化且美觀的 CSS 設計，營造優質的用戶體驗。

## 4. MVP 範圍

**Must Have (必須有)**
- 顯示當期應繳房租資訊。
- 線上繳費表單（模擬信用卡付款）。
- 寫入 SQLite 並儲存繳費紀錄。
- 歷史繳費紀錄列表。
- 測試用的 Mock Data（自動建立 User 與 Lease）。

**Should Have (應該有)**
- 防詐騙的安全提示 Modal 或醒目區塊。
- 響應式網頁設計（RWD），支援手機端操作。

**Nice to Have (可以有)**
- PDF 收據下載功能。
- 繳費成功後的 Email 或系統通知。
