# PRD: 租屋網站 - 爭議仲裁功能

## 1. 專案概述
- **背景與動機**：租屋常發生入住點交與退租時的屋況認知差異，導致押金扣抵爭議。需要平台作為第三方介入仲裁。
- **目標用戶**：平台管理員（主要操作者）、房東、房客（爭議當事人）。
- **核心價值主張**：透過客觀的照片證據與對比，提供公正、快速的初步裁決，保障雙方權益並減少糾紛。

## 2. 功能需求
- 作為 **房客/房東**，我希望 **能在訂單/合約發生問題時發起爭議仲裁**，以便 **讓平台介入處理**。
- 作為 **房東/房客**，我希望 **能上傳入住時的點交照片與退租時的屋況照片**，以便 **作為仲裁的證據**。
- 作為 **平台管理員**，我希望 **能有專屬介面檢視特定爭議案件中雙方上傳的所有照片（入住與退租）**，以便 **進行對比與判斷**。
- 作為 **平台管理員**，我希望 **能填寫並送出初步裁決結果與說明**，以便 **通知雙方裁決結果**。
- 作為 **房客/房東**，我希望 **能查看初步裁決的結果與管理員說明**，以便 **了解後續處理方式（如押金退還比例）**。

## 3. 非功能需求
- **技術限制**：使用 Flask + Jinja2 + SQLite。
- **效能與安全考量**：
  - 照片上傳需限制檔案類型（如 JPG, PNG）與大小，避免伺服器負載過大。
  - 權限控管：只有該案件的當事人（房東、房客）以及平台管理員可以存取這些照片與案件資料。

## 4. MVP 範圍
- **Must Have**: 雙方發起爭議、雙方上傳入住/退租照片功能、管理員檢視爭議案件與照片介面、管理員送出裁決結果。
- **Should Have**: 爭議案件列表（管理員待辦清單）、裁決狀態追蹤。
- **Nice to Have**: 照片加上時間戳記防偽、系統自動推播通知。

## 5. 專案成員與分工
- 爭議仲裁功能：由平台管理員（AI Agent 輔助）與開發者共同負責。
