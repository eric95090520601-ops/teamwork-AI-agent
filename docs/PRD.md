# PRD: 租屋網站 - 房東基本資料透明化(真房東認證)功能

## 1. 專案概述
- **背景與動機**：近年租屋詐騙頻傳，假房東騙取押金事件層出不窮。為了提升平台信任度，保障租客權益，平台需建立「真房東」認證機制，確保物件刊登者具備合法出租資格。
- **目標用戶**：房東（上傳認證資料）、租客（查看認證標章）、平台管理員（審核認證資料）。
- **核心價值主張**：透過實名認證與房屋產權查驗，過濾不實房源與詐騙份子，打造安心、透明的租屋環境。

## 2. 功能需求
- 作為 **房東**，我希望 **在刊登物件前能上傳身分證與房屋權狀影本（或仲介證照）**，以便 **申請真房東認證**。
- 作為 **房東**，我希望 **能在個人介面追蹤認證審核進度（如：審核中、已通過、已退件）**，以便 **隨時掌握認證狀態**。
- 作為 **平台管理員**，我希望 **有一個後台介面能審核房東上傳的認證文件**，以便 **確認其真實性並核發或拒絕認證**。
- 作為 **系統**，我希望 **能在房東通過認證後，自動在其刊登的物件上顯示「真房東認證」標章**，以便 **讓租客辨識**。
- 作為 **租客**，我希望 **在瀏覽房屋物件時，能清楚看到房東是否通過「真房東認證」**，以便 **作為租屋決策的參考**。

## 3. 非功能需求
- **技術限制**：使用 Flask + Jinja2 + SQLite。
- **效能與安全考量**：
  - 上傳的身分證與權狀影本屬於高度敏感個資，必須安全儲存，且除了審核管理員外不可對外公開。
  - 上傳圖片需限制檔案大小與格式（如 JPG, PNG, PDF 等）。
  - 自動在圖片上壓浮水印（例如「僅供租屋平台審核使用」），防止個資遭盜用。

## 4. MVP 範圍
- **Must Have**: 
  - 房東上傳身分證與房屋權狀/仲介證照介面。
  - 平台管理員審核介面（核准/拒絕）。
  - 認證狀態關聯到房東資料。
  - 前台物件頁面與房東資料顯示「真房東認證」標章。
- **Should Have**:
  - 審核退件時可填寫退件理由。
  - 房東上傳文件時，系統自動加上防偽浮水印（保護隱私）。
- **Nice to Have**: 
  - 介接政府 API 進行自動驗證（取代人工審核）。
  - 認證狀態變更時發送 Email/系統通知給房東。
