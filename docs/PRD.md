# PRD: 租屋網站 - 爭議仲裁與報修溝通中心

## 1. 專案概述
- **背景與動機**：
  1. 租屋常發生入住點交與退租時的屋況認知差異，導致押金扣抵爭議。需要平台作為第三方介入仲裁。
  2. 租屋期間若有家具損壞，經常因缺乏報修紀錄與原始狀態對比而引發糾紛，需要「報修與溝通中心」留存對話與證據。
- **目標用戶**：平台管理員（主要操作者）、房東、房客（爭議與報修當事人）。
- **核心價值主張**：透過客觀的照片/影片證據與對比，提供公正的裁決與報修紀錄，保障雙方權益並減少糾紛。

## 2. 功能需求
- 作為 **房客/房東**，我希望 **能在訂單/合約發生問題時發起爭議仲裁**，以便 **讓平台介入處理**。
- 作為 **房東/房客**，我希望 **能上傳入住時的點交照片與退租時的屋況照片**，以便 **作為仲裁的證據**。
- 作為 **平台管理員**，我希望 **能有專屬介面檢視特定爭議案件中雙方上傳的所有照片（入住與退租）**，以便 **進行對比與判斷**。
- 作為 **平台管理員**，我希望 **能填寫並送出初步裁決結果與說明**，以便 **通知雙方裁決結果**。
- 作為 **房客/房東**，我希望 **能查看初步裁決的結果與管理員說明**，以便 **了解後續處理方式（如押金退還比例）**。

### 報修與溝通中心功能
- 作為 **房客/房東**，我希望 **能發起報修單並描述損壞物品**，以便 **通知對方進行處理**。
- 作為 **房客/房東**，我希望 **能上傳損壞物品的原本狀態與損壞狀態之照片/影片**，以便 **進行對比，避免損壞程度的認知糾紛**。
- 作為 **房客/房東**，我希望 **能在報修單內擁有留言板功能**，以便 **進行溝通，且所有紀錄皆留存在平台上作為證據**。

## 3. 非功能需求
- **技術限制**：使用 Flask + Jinja2 + SQLite。
- **效能與安全考量**：
  - 照片上傳需限制檔案類型（如 JPG, PNG）與大小，避免伺服器負載過大。
  - 權限控管：只有該案件的當事人（房東、房客）以及平台管理員可以存取這些照片與案件資料。

## 4. MVP 範圍
- **Must Have**: 
  - 爭議仲裁：雙方發起爭議、上傳照片、管理員裁決。
  - 報修溝通：發起報修、上傳原本/損壞照片對比、報修專屬留言板。
- **Should Have**: 爭議/報修列表、狀態追蹤。
- **Nice to Have**: 支援影片上傳（限制檔案大小）、自動推播通知。

## 5. 專案成員與分工
- 爭議仲裁與報修溝通功能：由平台管理員（AI Agent 輔助）與開發者共同負責。
